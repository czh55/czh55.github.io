分支的原理和本质

廖海峰 分支实现原理和本质

  1. HEAD指向的是当前分支，而不是一直指向的master分支
  2. Git创建一个分支很快，因为除了增加一个dev指针，改改HEAD的指向，工作区的文件都没有任何变化！
  3. 只在一个分支dev上修改代码，Git合并分支也很快！就改改指针，工作区内容也不变
  4. 在多个分支上都修改代码，每个分支有不同的代码，切换分支idea会自动切换代码
